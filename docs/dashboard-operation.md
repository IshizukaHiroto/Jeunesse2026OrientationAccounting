# ダッシュボード運用手順

図で全体を確認したい場合は、先に [`visual-guide.md`](./visual-guide.md) を参照してください。

## 1. 初期設定
1. [`assets/config.js`](../assets/config.js) の `GAS_ENDPOINT` を本番URLに更新。
2. `POLLING_MS` は既定値 `60000`（60秒）を維持。
3. CSS更新時は `npm run build:css` を実行して [`assets/styles.css`](../assets/styles.css) を再生成。

## 2. ローカル確認
1. `npm install`
2. `npm run build:css`
3. `npm test`
4. 静的サーバーで表示確認（例: `npx serve .`）

## 3. GitHub Pages公開
1. `main` ブランチへ反映。
2. `Settings > Pages` の公開対象を `main` に設定。
3. 公開URLでスマホ表示（幅390px）を検証。

## 4. 日常運用
1. 申請承認後、最大60秒で反映。
2. 取得失敗時は黄色バナーで前回成功データ表示に切り替わる。
3. 返金完了後は `支出` タブの `立替` フィルターで反映を確認する。
4. 公開UIには管理者向けの確認先を出さず、詳細な障害切り分けは運用手順で行う。

## 5. 画面操作
1. PCでは左サイドバー、スマホでは下部タブで `サマリー / 収入 / 支出` を切り替える。
2. `サマリー` は4つのKPIカード、`全体収支状況`、`予算使用状況` を上から確認する。`総支出額` と各 `支出額` には `経費 + 立替予定額` が反映される。
3. `収入` は `氏名 / 金額 / 納入日 / 状態` の一覧リストを表示し、`フィルター` から `昇順 / 降順` と `納入済みだけ表示` を切り替える。
4. `支出` は `氏名 / 種別 / 項目 / 金額 / 日付 / 返金状況` を表示し、`フィルター` から `すべて / 経費 / 立替` を切り替える。`経費` の `氏名` は `新歓係`、`返金状況` は `--`、`立替` は公開JSONの値を表示する。
5. 立替の返金確認は `支出` タブの `立替` フィルターで行う。専用の `返金状況` タブは使わない。
6. 更新中は通常の `更新` ボタンが `更新中...` の非クリック表示に切り替わり、完了後に通常ボタンへ戻る。独立した `更新中` / `更新済み` ラベルは表示しない。
7. 一覧は初期5件から表示され、`もっと見る` を押すと全件表示される。全件表示後にもう一度押すと `折りたたむ` として初期表示に戻る。

## 6. 障害時対応
1. バナー表示が続く場合、管理者はまずGAS URLと公開権限を確認する。公開UIにはこれらの内部確認先を表示しない。
2. シート名変更時は [`gas/Code.gs`](../gas/Code.gs) の候補名を更新。
3. JSON契約変更時は `docs/gas-deploy-and-json-contract.md` とテストを同時更新。
