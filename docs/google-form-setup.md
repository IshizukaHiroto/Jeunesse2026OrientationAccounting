# Googleフォーム設定手順

## 1. フォーム作成
1. Googleフォームを新規作成。
2. タイトル例: `新歓立替申請フォーム（2026）`。

## 2. 設問定義
以下の順で設定する。

1. `申請日`（日付）
2. `立替者ニックネーム`（プルダウン）
3. `内容`（記述式）
4. `支払い金額`（記述式: 数値）
5. `新入生の人数`（記述式: 数値）
6. `レシート画像`（ファイルアップロード）

## 2-1. 列名の互換について
フォーム回答の見出しが次のような名前でも、GASは自動で読み取れます。

- `申請者名`
- `申請者名(上記に名前がない場合)`
- `内容（記述）`
- `立替金額の合計(数字のみ記載)`
- `立て替えた新入生の合計人数`

`立替者ニックネーム` と表記が違っていても、`申請者名` 系の列が埋まっていればダッシュボードで名前表示されます。

## 3. ニックネーム選択肢
1. `集金管理` のニックネーム列を参照して候補を管理する。
2. 候補更新後はフォーム側選択肢を同期更新する。
3. 名前がない場合の連絡先をフォーム説明文に書く。

## 4. 回答先シート連携
1. フォームの `回答` タブを開く。
2. `スプレッドシートにリンク` を選択。
3. 対象ファイルとして会計シートを指定し、返金申請の回答先タブは `立替返金管理` を推奨名として使う。
4. Google フォーム連携直後に `フォームの回答 1` または `Form Responses 1` が作られた場合も、GAS は1枚だけなら自動認識する。ただし複数あると曖昧になるため、運用上は `立替返金管理` にリネームする。

## 5. ファイルアップロードの注意
1. レシートアップロードは回答者のGoogleログインが必要。
2. ログイン不可時の代替手段（LINE送付）をフォーム説明に明記。
3. レシートURLは外部公開JSONに含めない。

## 6. 締切運用
1. 最終イベント後1週間で申請締切。
2. 締切後の回答は削除せず、シート上で `無効` に変更して理由へ `締切後申請` と記録。
