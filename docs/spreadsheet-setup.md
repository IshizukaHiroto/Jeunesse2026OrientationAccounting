# スプレッドシート設定手順（2026年度）

## 1. 構成
1. 新しいGoogleスプレッドシートを作成し、以下4シートを作る。
- `集金管理`
- `立替返金管理`
- `経費記録`
- `設定`

## 2. 設定シート（`設定`）
A列に項目名、B列に値を入れる。

| A列 | B列 | 備考 |
| --- | --- | --- |
| 集金額（1人あたり） | 4000 | 円 |
| 返金上限（新入生1人あたり） | 700 | 円 |
| 新歓期間（開始） | 2026/04/01 | 任意変更可 |
| 新歓期間（終了） | 2026/05/31 | 任意変更可 |

## 3. 集金管理シート（`集金管理`）
### 列定義
| 列 | 項目 | 入力 |
| --- | --- | --- |
| A | ニックネーム | 手入力 |
| B | 支払い状況 | プルダウン（`済`,`未`） |
| C | 集金額 | 自動式 |
| D | 確認日 | 日付 |
| E | 備考 | 任意 |

### 推奨式
- `C2`: `=IF(B2="済",設定!$B$1,0)`

## 4. 立替返金管理シート（`立替返金管理`）
### 列定義
| 列 | 項目 | 入力 |
| --- | --- | --- |
| A | 申請日 | フォーム連携 |
| B | 立替者ニックネーム | フォーム連携 |
| C | 内容 | フォーム連携 |
| D | 支払い金額 | フォーム連携 |
| E | 新入生の人数 | フォーム連携 |
| F | 返金額 | 自動式 |
| G | 承認状況 | プルダウン（`承認済`,`未承認`） |
| H | 返金状況 | プルダウン（`返金済`,`未返金`） |
| I | 無効フラグ | プルダウン（`有効`,`無効`） |
| J | 無効理由 | 手入力 |

### シート名の扱い
- 推奨シート名は `立替返金管理`。
- Googleフォーム連携直後の既定名 `フォームの回答 1` / `Form Responses 1` も、該当タブが1枚だけならGASが自動認識する。
- 既定名の回答タブが複数ある場合は判別できないため、返金申請用タブを `立替返金管理` にリネームする。

### 列名互換（重要）
GASは以下の表記ゆれを吸収します。列名を完全一致させなくても読み取れます。

- `立替者ニックネーム` / `立替者(ニックネーム)` / `立替者（ニックネーム）`
- `申請者名`
- `申請者名(上記に名前がない場合)` / `申請者名（上記に名前がない場合）`
- `内容` / `内容(記述)` / `内容（記述）`
- `支払い金額` / `立替金額の合計(数字のみ記載)`
- `新入生の人数` / `立て替えた新入生の合計人数`
- `申請日` / `日付`

### 推奨式
- `F2`: `=MIN(D2,E2*設定!$B$2)`

### 運用ルール
1. 行削除は禁止。
2. 誤申請や締切後申請は `無効フラグ=無効` + `無効理由` を記録。
3. ダッシュボード集計対象は `承認済` かつ `有効` のみ。

## 5. 経費記録シート（`経費記録`）
| 列 | 項目 | 入力 |
| --- | --- | --- |
| A | 日付 | 手入力 |
| B | 内容 | 手入力 |
| C | 金額 | 手入力 |
| D | 支払い者 | 手入力 |
| E | 備考 | 任意 |

## 6. データ検証（プルダウン）
1. `集金管理!B:B` に `済,未` を設定。
2. `立替返金管理!G:G` に `承認済,未承認` を設定。
3. `立替返金管理!H:H` に `返金済,未返金` を設定。
4. `立替返金管理!I:I` に `有効,無効` を設定。

## 7. 年度更新ルール
1. 年度が変わる場合は `設定` シートの期間を更新。
2. ダッシュボード文言と `gas/Code.gs` の `seasonYear` も同時更新。
3. 変更内容は `docs` に同一コミットで反映する。
