# 受け入れチェックリスト

## 機能チェック
- [ ] 返金額が `MIN(支払額, 新入生人数 × 上限)` で計算される。
- [ ] 残高プラス時に均等割り合計が残高と一致し、端数は最後の1件のみ調整される。
- [ ] 残高マイナス時に按分合計が利用可能額と一致し、端数は最大返金1件のみ調整される。
- [ ] `承認済` かつ `有効` 以外の立替が表示されない。
- [ ] レシートURL・未承認データ・無効データがJSONに含まれない。
- [ ] 60秒ポーリングで自動反映される。
- [ ] 取得失敗時に警告バナーが表示され、前回成功データを維持する。

- [ ] スマホ幅390pxで横スクロールが発生しない。
- [ ] スマホ幅390pxで下部タブバーが表示される。
- [ ] 下部タブから「サマリー / 集金 / 出費」が切り替わる。
- [ ] 初期表示で「サマリー」「残高案内」「最終更新」が先頭に見える。
- [ ] 「集金進捗」と「残高内訳」の2つの円グラフがサマリー画面に表示される。
- [ ] 残高内訳グラフで出費/残高が色分けされ、出費内訳として「経費 / 立替返金予定」が読める。
- [ ] 残高が赤字のとき、不足分が外側の赤リングで表示される。
- [ ] 集金一覧は名前の昇順/降順で並び替えできる（初期は昇順）。
- [ ] 出費画面で「経費明細」と「立替一覧（承認済・有効のみ）」を続けて確認できる。
- [ ] 立替一覧は名前の昇順/降順で並び替えできる（初期は昇順）。
- [ ] 集金一覧と出費画面内の経費・立替一覧は初期5件表示で、`もっと見る` で段階表示できる。
- [ ] スマホ幅390pxで下部タブと一覧の `もっと見る` が指で押しやすいサイズになっている。
- [ ] スマホで更新中表示が1行に収まり、ボタン高さが不自然に増えない。
- [ ] 更新中はボタンが「更新中...」表示になり、読み込み案内メッセージが表示される。
- [ ] サマリー画面の精算ルールが初見ユーザー向けの平易な説明になっている。
- [ ] 返金上限（新入生1人あたり）がサマリー精算ルールと出費画面の立替欄説明の2か所に表示される。
- [ ] `meta.refundCapPerFreshman` が有効値ならその金額を表示し、欠損・不正時は700円を表示する。
- [ ] 精算ルール文面に内部運用情報や公開不要な手順が含まれない。
