# 受け入れチェックリスト

## 機能チェック
- [ ] 返金額が `MIN(支払額, 新入生人数 × 上限)` で計算される。
- [ ] 残高プラス時に均等割り合計が残高と一致し、端数は最後の1件のみ調整される。
- [ ] 残高マイナス時に按分合計が利用可能額と一致し、端数は最大返金1件のみ調整される。
- [ ] `承認済` かつ `有効` 以外の立替が表示されない。
- [ ] レシートURL・未承認データ・無効データがJSONに含まれない。
- [ ] `collection.note`、`expenses.category`、`expenses.payer`、`expenses.note`、`reimbursements.approvalStatus`、`reimbursements.invalidFlag` などの公開不要キーがJSONに含まれない。
- [ ] 60秒ポーリングで自動反映される。
- [ ] 取得失敗時に警告バナーが表示され、前回成功データを維持する。

- [ ] スマホ幅390pxで下部タブバーが表示される。

- [ ] 初期表示で4つのKPIカードと「全体収支状況」「予算使用状況」が先頭に見える。

## ダブルチェック（AIっぽさ対策）
### 文言監査
- [ ] 曖昧表現（例: 「適切に」「必要に応じて」）を避け、操作動詞で記述している。
- [ ] 説明文が過剰に長くなく、1文1意図で読める。
- [ ] 公開UIの文言に `GAS`、`JSON`、`公開設定`、`外側赤リング` などの内部用語が出ない。
- [ ] 更新状態ラベルが `同期OK` や `同期待機` ではなく、利用者向けの自然な表現になっている。
- [ ] ダッシュボード上に `1人あたりの返金目安` と `返せる割合` が表示されない。

### UI監査
- [ ] 典型テンプレ風の無機質レイアウトを避け、用途に沿った階層がある。
- [ ] 配色は状態判別に使い、色以外のラベル（済/未返金など）も併記している。
- [ ] 情報密度が中程度で、一覧が詰まりすぎていない。
- [ ] 画面全体で常時アニメーションを使っていない。
- [ ] favicon 404 など、公開ページで不要なコンソールエラーが出ない。
