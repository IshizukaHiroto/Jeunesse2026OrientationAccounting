設計仕様書・運用マニュアル

**【完全最終版 v3】**

対象者：新歓係・サークル員全員

# **1\. システム概要**

## **1-1. 目的**

テニスサークルの新歓活動（4月上旬〜5月末）にかかる費用を透明性高く管理するためのシステムです。新歓係が唯一の管理者となり、Googleスプレッドシートでデータを管理します。その内容を自動でウェブサイト（ダッシュボード）に反映し、サークル員全員がいつでも現状を確認できるようにします。

## **1-2. 登場人物**

| 役割 | できること | できないこと |
| ----- | ----- | ----- |
| 新歓係（管理者） | スプレッドシートの編集・承認・無効化 | — |
| サークル員（閲覧者） | ダッシュボードの閲覧 | スプレッドシートの編集 |
| 立替者 | Googleフォームで申請・レシート提出 | スプレッドシートの直接編集 |

## **1-3. お金の流れ**

① サークル員全員から集金（1人あたり4,000円、変更可能）

② 新歓イベント・食事などで誰かが立替払い

③ 立替者がGoogleフォームでレシートと一緒に申請

④ 新歓係が内容確認・承認をスプレッドシートに記録

⑤ 新歓期間終了後（最後のイベントから1週間以内に申請締切）

⑥ 5月末に一括精算：立替者へ返金 ＋ 残高があれば全員に均等割り返金

## **2-1. 返金計算ルール（具体例）**

| ケース | 支払額 | 新入生人数 | 返金額 |
| ----- | ----- | ----- | ----- |
| 新入生1人に500円の飲み物 | 500円 | 1人 | 500円（実費） |
| 新入生1人に1,000円の食事 | 1,000円 | 1人 | 700円（上限適用） |
| 新入生3人で1,800円の食事 | 1,800円 | 3人 | 1,800円（1人600円 \< 700円） |
| 新入生3人で3,000円の食事 | 3,000円 | 3人 | 2,100円（3人×700円上限） |

*※ 返金額 \= MIN(支払額, 新入生人数 × 700\) の計算式をスプレッドシートで自動算出します。*

## **2-2. 残高マイナス時の按分計算と端数処理（具体例）**

例：返金予定総額 50,000円 ／ 集金残高 40,000円 ／ 立替者3名

| 立替者 | 本来の返金額 | 按分後（切り捨て） | 最終返金額 |
| ----- | ----- | ----- | ----- |
| Aさん（最大返金額） | 20,000円 | 16,000円 | 16,200円（端数+200円を負担） |
| Bさん | 15,000円 | 12,000円 | 12,000円 |
| Cさん | 15,000円 | 12,000円 | 12,000円（←最後の1件ではない） |
| 合計 | 50,000円 | 40,000円未満 | 40,000円（残高と一致） |

*※ 按分率 \= 集金残高 ÷ 返金予定総額。各自は切り捨て、端数は返金額が最大の人の1件に上乗せして残高を0にします。*

## **2-3. 残高プラス時の均等割り返金と端数処理（具体例）**

例：精算後の残高 6,100円 ／ 集金済みメンバー 15人

6,100 ÷ 15 \= 406.6… → 切り捨て400円 × 15人 \= 6,000円

余り100円 → 最後の1件（例：返金処理の最後の人）に+100円して総額6,100円を返金

*※ 均等割りした金額を全員に返金し、端数は最後の1件に上乗せして残高を必ず0にします。*

## **2-4. 予算サマリーの算出式【明文化】**

| 表示項目 | 算出式 | 集計対象 |
| ----- | ----- | ----- |
| 集金総額 | 集金済み人数 × 集金額（設定値） | Sheet1：支払い状況＝「済」の行 |
| 経費合計 | 全経費の金額合計 | Sheet3：全行 |
| 返金予定額 | 承認済・有効な立替申請の返金額合計 | Sheet2：承認済＝「承認済」かつ無効フラグ＝「有効」の行 |
| 現在残高 | 集金総額 − 経費合計 − 返金予定額 | 上記3項目から自動計算 |
| 均等割り返金額 | 現在残高 ÷ 集金済み人数（残高プラス時のみ表示） | 残高プラス時のみ |
| 按分率 | 現在残高 ÷ 返金予定額（残高マイナス時のみ表示） | 残高マイナス時のみ |

**⚠ 残高がマイナスの場合はダッシュボードに赤色で警告表示し、按分率を明示します。**

# **4\. Googleスプレッドシート設計**

スプレッドシートは4枚のシートで構成します。

## **Sheet 1：集金管理**

| 列名 | 内容 | 入力方法 |
| ----- | ----- | ----- |
| ニックネーム | サークル員の表示名（ダッシュボードに表示・フォームのプルダウン元） | 手入力 |
| 支払い状況 | 済 / 未 | プルダウン選択 |
| 集金額 | 設定シートの値を参照（自動） | 自動計算 |
| 確認日 | 振り込みを確認した日付 | 手入力 |
| 備考 | メモがあれば記入 | 手入力 |

*※ このシートのニックネーム列がGoogleフォームのプルダウン選択肢の元データになります。*

*※ 集計セル：集金済み人数・集金残高・未払い者リストを関数で自動計算します。*

## **Sheet 2：立替・返金管理**

| 列名 | 内容 | 入力方法 |
| ----- | ----- | ----- |
| 申請日 | 立替が発生した日付 | フォームから自動入力 |
| 立替者ニックネーム | Sheet1のプルダウンから選択した表示名 | フォームから自動入力 |
| 内容 | 何に使ったか | フォームから自動入力 |
| 支払い金額 | 実際に払った金額（円） | フォームから自動入力 |
| 新入生の人数 | 奢った新入生の人数 | フォームから自動入力 |
| 返金額 | MIN(支払額, 人数×700)を自動計算 | 自動計算 |
| 承認状況 | 承認済 / 未承認 | 新歓係がプルダウン選択 |
| 返金状況 | 返金済 / 未返金 | 新歓係がプルダウン選択 |
| 無効フラグ | 有効 / 無効 | 新歓係がプルダウン選択（誤申請時） |
| 無効理由 | 無効にした理由 | 手入力（無効時のみ） |

*※ ダッシュボードの集計対象は「承認済」かつ「有効」の行のみです。*

*※ 行の削除は禁止。誤申請は「無効フラグ」を「無効」にして理由を記入してください。*

## **Sheet 3：経費記録**

| 列名 | 内容 | 入力方法 |
| ----- | ----- | ----- |
| 日付 | 支出が発生した日付 | 手入力 |
| 内容 | 何に使ったか | 手入力 |
| 金額 | 支出金額（円） | 手入力 |
| 支払い者 | 実際に払った人 | 手入力 |
| カテゴリ | イベント費 / 消耗品 / 食費 など | プルダウン選択 |
| 備考 | メモがあれば記入 | 手入力 |

*※ 立替（Sheet2）以外の直接的な経費をここに記録します。ダッシュボードの利用明細に表示されます。*

# **5\. ウェブサイト（ダッシュボード）仕様**

## **5-1. 画面構成**

| エリア | 表示内容 | データソース |
| ----- | ----- | ----- |
| ヘッダー | サークル名・タイトル・GAS JSON生成日時 | GAS |
| 左：円グラフ | 集金状況（済 / 未）の割合と現在残高 | Sheet1 |
| 右：利用明細 | 経費の一覧（日付・内容・金額・カテゴリ） | Sheet3 |
| 下：立替一覧 | 承認済・有効な立替申請の一覧と返金状況 | Sheet2 |
| 下：予算サマリー | 集金総額・経費合計・返金予定額・現在残高・均等割り/按分情報 | 全シート |

## **5-2. 表示ルール**

* 支払い状況「済」→ 緑色で表示

* 支払い状況「未」→ 赤色で表示

* 返金状況「返金済」→ グレーで表示

* 返金状況「未返金」→ オレンジで表示

* 残高がマイナスの場合 → 赤色警告 ＋ 按分率を表示

* 残高がプラスの場合 → 緑色 ＋ 1人あたり均等割り返金額を表示

## **5-3. JSONに含めるデータ・含めないデータ**

| データ | JSON | 理由 |
| ----- | ----- | ----- |
| 集金状況（ニックネーム・済未） | ◎ 含める | ダッシュボード表示に必要 |
| 立替情報（承認済・有効のみ） | ◎ 含める | ダッシュボード表示に必要 |
| 経費記録 | ◎ 含める | 利用明細表示に必要 |
| 未承認の立替データ | ✕ 含めない | 確認前の情報を公開しない |
| 無効フラグが立ったデータ | ✕ 含めない | 誤申請情報を公開しない |
| レシート・領収書の画像URL | ✕ 含めない | 外部公開不要・プライバシー保護 |
| サークル員の本名・連絡先 | ✕ 含めない | 個人情報保護 |

# **7\. 構築手順**

## **Step 1：Googleスプレッドシートを作成**

* Googleドライブで新しいスプレッドシートを作成する

* シートを4枚作成し「集金管理」「立替返金管理」「経費記録」「設定」と名付ける

* 各シートに列名を入力し、設定シートに集金額（4000）と返金上限（700）を記入する

* 支払い状況・承認状況・返金状況・無効フラグの列にプルダウンを設定する

## **Step 2：Googleフォームを作成・Sheet2と連携**

* Googleフォームで新しいフォームを作成する

* 項目を追加：日付 / ニックネーム（プルダウン：Sheet1のニックネーム列を参照） / 内容 / 支払い金額 / 新入生の人数 / レシート画像（ファイルアップロード）

* フォームの「回答」タブ → スプレッドシートと連携 → Sheet2を指定

* フォームのURLをサークル員にLINEで共有する

## **Step 3：Google Apps Script（GAS）を設定**

* スプレッドシートの「拡張機能」→「Apps Script」を開く

* Claudeが生成したGASコードを貼り付ける

* 「デプロイ」→「新しいデプロイ」→「ウェブアプリ」として公開

* アクセス権限を「全員（Googleアカウント不要）」に設定し、URLをコピーする

## **Step 4：GitHub PagesでダッシュボードをWeb公開**

* GitHubリポジトリ（Jeunesse2026OrientationAccounting）にindex.htmlをアップロード

* index.html内のGAS URLを自分のURLに書き換える

* 「Settings」→「Pages」→「Branch: main」を選択して保存

* 数分後に発行されたURLをサークル員にLINEで共有する

*※ GitHubへのファイルアップロードはブラウザ上でドラッグ＆ドロップでできます。コマンドライン不要です。*

# **8\. よくある質問（FAQ）**

| 質問 | 回答 |
| ----- | ----- |
| 集金額が4,000円から変わった場合は？ | Sheet4（設定）の集金額を変更するだけで全シートに自動反映されます。 |
| 立替申請を間違えて送ってしまった場合は？ | Sheet2の該当行の「無効フラグ」を「無効」にして「無効理由」を記入してください。行は削除しません。 |
| 残高がマイナスになったらどうなりますか？ | 追加集金はしません。立替返金額を全員で按分して削ります。各自切り捨て、端数は最後の1件で調整して残高を0にします。 |
| 残高がプラスで余ったらどうなりますか？ | 集金済み人数で均等割りして全員に返金します。各自切り捨て、端数は最後の1件に上乗せして残高を0にします。 |
| スプレッドシートを更新してからダッシュボードに反映されるまでの時間は？ | ページを再読み込みすれば即時反映されます。 |
| 外部の人にダッシュボードが見られても問題ないですか？ | 個人情報・レシート画像・未承認データはJSONに含めないため問題ありません。ニックネームと金額は見られます。 |
| 申請締め切り後に申請が来た場合はどうすればよいですか？ | 「無効フラグ」を「無効」にして「無効理由」に「締切後申請」と記入してください。 |
| Googleフォームでレシートがアップロードできない場合は？ | Googleフォームのファイルアップロードにはレシート提出者のGoogleアカウントへのログインが必要です。ログインできない場合や容量オーバーの場合は、LINEで新歓係に画像を直接送信してください。 |
| フォームでニックネームを選ぶプルダウンに自分の名前がない場合は？ | 新歓係にLINEで連絡し、Sheet1にニックネームを追加してもらってから再度フォームを送信してください。 |
| 新歓期間が終了したらスプレッドシートはどうすればよいですか？ | 精算完了後もそのまま保存して記録として残してください。翌年度の参考にもなります。 |
